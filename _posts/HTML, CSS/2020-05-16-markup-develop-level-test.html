HTML

1. Doctype을 사용하지 않을 때는 무슨 일이 발생할까요?

doctype은 html버전을 선언한다. chrome에서는 html5으로 인식한다.

2. 웹표준에 맞게 작업할 때 b, i 태그 대신 적합한 태그는 각각 무엇일까요?

<strong> , <em>

3. block 요소와 inline 요소에 해당하는 태그들을 각각 5개씩 적어보세요.

block 요소: <div>, <p>, <blockquote>, <hn>, <ul>
inline 요소: <strong>, <em>, <q>, <small>, <a>

4. blockquote 태그는 어떤 용도로 사용해야 할까요?

2줄이상의 인용문

5. 인라인 스타일(style=“property:value”)을 가급적 사용하지 말아야 할 이유는 무엇일까요?

style은 외부 !important, inline style, <style>, css 순서로 적용된다. PlugIn이나 javascript에서 적용하는 스타일은 inline이기 때문에 서로 충돌할 수 있다.
서치결과: css 유지관리의 불편함, 시멘틱 웹에 부적합(웹표준 기본 요구조건: 구조와 표현의 분리), html 정보량이 많을 수록 효율 저하

6. myPhoto.jpg 파일을 img 태그로 작성해보세요.

<img src="myPhoto.jpg alt="">

7. HTML에서 id 속성을 사용하는 이유와 주의 할 점은 무엇일까요?

html 문서 내에서 동일한 이름의 id는 2번 사용할 수 없다. class보다 id가 우선순위가 높으므로 신중해야한다.

8. ‘TopArea'라는 클래스명이 좋지 않은 이름이라면 그 이유는 무엇일까요?

class명은 소문자로 시작하는 것이 좋다. class 명은 표현형보다는 의미를 포함하는 것이 좋다.

9. 'blue-box'라는 클래스명이 좋지 않은 이름이라면 그 이유는 무엇일까요?

class 명은 표현형보다는 의미를 포함하는 것이 좋다.

10. HTML5에 새롭게 추가된 aside 태그는 어떤 용도로 사용해야 할까요?

메뉴, 광고
서치 결과: 컨텐츠 주요 내용이 아닌 부수적 내용

11. input 태그와 항상 함께 사용해야 할 태그는 무엇일까요?

<form>, <label>

12. 모바일 웹 또는 반응형웹디자인 프로젝트에서 각 기기에 적합한 화면을 보여주기 위해 필요한 meta 태그는 무엇일까요?

<meta name="viewport" content="width=device-width initial-scale=1.0, user-scalable=no”>

CSS

1. 화면 상에는 보이지 않으나, 스크린 리더에서 읽혀야 하는 요소에 주어야 할 스타일링을 작성해보세요.

1. opacity: 0;
2. text-indent: -9999;
3. color: transparent;
display: none;, visibility: hidden은 안됨. 자세한 내용은 "IR기법"으로 서치

2. float 속성을 가진 자식을 품은 부모요소는 높이 값이 0이 되는 때가 있습니다. 이 현상을 해소하는법(clearing)을 알고 있나요?

1. oeverflow: hideen;
2. font-size: 0;
3. ::after{content: ""; display: block; clear: both;}

3. border-box와 content-box의 차이점은 무엇일까요?

border-box: width에 padding, border-width 포함
content-box: width에 padding, border-width 미포함

4. position: relative는 어떤 경우에 사용 하나요?

자식요소가 position: absolute일 때 기준값을 잡아줄 때, 레이아웃은 변경하지 않고 위치를 조절 할 때
서치결과: 좌표 재정의, 쌓임 순서 지정

5. CSS Reset은 무엇이며 왜 사용할까요?

브라우저에서 태그에 기본적으로 적용하는 css 중 일부를 없애기 위해 사용한다.
서치결과: 크로스브라우징을 목적으로

6. Sass, less, Stylus와 같은 CSS 프리프로세서를 사용해본적이 있나요?

아니요

7. id, class, inline style, !important를 우선순위 순으로 나열해보세요.

!important, inline style, id, class

8. CSS에서 상속이 되는 속성을 2개만 꼽아보세요.

font-family, font-size
서치결과: color, font-size, font-family, letter-spacing

9. Sprite image 기법을 사용하는 이유는 무엇일까요?

이미지를 한장만 불러오면 되기 때문에 load 속도가 빨라진다.
서치결과: 파일 요청횟수 감소, 이미지 크기 감소

10. Sprite image 기법을 사용하는데 필요한 CSS 속성들을 꼽아보세요.

background-image, backgroud-position

11. 점진적 향상(Progressive Enhancement)의 뜻을 설명 할 수 있나요?

모름

12. 웹폰트를 적용하기 위해서는 어떤 확장자의 폰트 파일들이 필요할까요?

eot, tff, woff

13. 개발사 접두어(vendor prefix)를 꼭 사용해야 할 CSS 속성(property)를 2개만 꼽아보세요.

scrollbar. border-radius, transition, transform은 최근 표준화되어 vender prefix 불필요

14. 반응형웹디자인의 3가지 요소를 꼽아보세요.

서치결과: grid, plexible, user-interaction

15. 모바일기기를 대응할 때 기준으로 삼는 해상도 사이즈는 몇이며 그 이유는 무엇인가요?

960px(태블릿), 720px(태블릿), 480px(핸드폰), 320px(핸드폰)을 기본으로 하나, 레이아웃을 최우선 기준으로 삼는다.

16. :first-child와 :last-child가 지원하는 IE의 버전명을 적어보세요.

IE 7 이상, IE 9 이상(예외처리를 first-child로 하는 이유)

17. 포토샵(또는 다른 그래픽툴)에서 이미지를 자를 때 어떤 기능을 사용하나요? (메뉴명, 단축키 등)

slice(단축기 c)

18. jpg, gif, png의 차이점을 설명해보세요.

jpg: 투명화 불가능, 24bit-color, 손실압축
gif: 투명화 가능(단색 두명층), 8bit-color, 비손실압축(색상 때문에 손실되기도 함)
png: 투명화 가능(8bit 알파 채널), 24bit-color, 비손실압축

19. 가상컨텐츠(:before, :after)는 어떤 용도로 사용할까요?

html파일에 element를 새로 추가하지 않고 꾸밀 때 사용. jquery로 선택 불가

20. 블럭요소 안의 어떤 자식 요소를 정중앙에 놓는 방법을 알고 있습니까?

1. margin: [(부모요소 height - 자식요소 height)/2] auto;
2. posiotn: absolute; top: 50%; left: 50%; margin-left: -[(자식요소 width)/2]; margin-top: -[(자식요소 height)/2];(부모요소에 position: relative; 필요)
3. posiotn: absolute; top: [(부모요소 height - 자식요소 height)/2]; left: [(부모요소 width - 자식요소 width)/2];
4. posiotn: absolute; top: calc( 50% - [자식요소 height]/2); left: calc( 50% - [자식요소 width]/2);(IE transition에서 calc가 지원되지 않으니 가급적 사용하지 않음)
5. display: table-cell;(부모요소에 display: table; 필요)
